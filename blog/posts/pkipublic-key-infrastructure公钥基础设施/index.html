1. PKI概述

1.1 定义

PKI（Public Key Infrastructure，公钥基础设施）是一种基于公钥密码学的安全基础设施，用于创建、管理、分发、使用、存储和撤销数字证书。

1.2 核心目标

  • 身份认证：验证通信双方的身份

  • 数据完整性：确保数据在传输过程中未被篡改

  • 数据机密性：保证敏感信息的保密性

  • 不可否认性：确保发送者无法否认其发送的行为

1.3 发展历程

| 阶段 | 时间 | 里程碑 |

|——|——|——–|

| 萌芽期 | 1970s | Diffie-Hellman密钥交换算法诞生 |

| 理论奠基 | 1976 | RSA公钥密码系统提出 |

| 标准建立 | 1988 | X.509标准首次发布 |

| 商业化 | 1990s | SSL/TLS协议广泛部署 |

| 现代化 | 2000s-至今 | 移动互联网、IoT、区块链融合 |

2. 核心组件

2.1 证书颁发机构（CA）

定义：受信任的实体，负责颁发、管理和撤销数字证书。

CA层级结构：


根CA（Root CA）

    ├── 中间CA（Intermediate CA）

    │       ├── 颁发CA（Issuing CA）

    │       └── 策略CA（Policy CA）

    └── 自签名CA

CA职责：

  • 证书签发

  • 证书吊销管理

  • CRL（证书吊销列表）发布

  • OCSP（在线证书状态协议）响应

  • 策略制定与执行

2.2 注册机构（RA）

定义：负责证书申请者身份验证的机构，是CA的前端接口。

RA功能：

  • 接收并验证证书申请

  • 身份凭证审核

  • 密钥生命周期管理

  • 证书撤销请求处理

CA与RA的关系：


用户 → RA（身份验证）→ CA（证书签发）

2.3 证书存储库

功能：存储和分发证书及吊销信息。

组件类型：

| 组件 | 描述 |

|——|——|

| 证书存储 | X.500/LDAP目录存储证书 |

| CRL分发点 | 发布证书吊销列表 |

| OCSP响应器 | 提供在线证书状态查询 |

2.4 证书撤销列表（CRL）

定义：由CA签发的被撤销证书列表。

CRL字段：

  • 签名算法标识

  • 颁发者名称

  • 本次更新日期

  • 下次更新日期

  • 撤销证书列表

2.5 端实体

定义：使用PKI服务的最终用户、设备或应用程序。

类型：

  • 个人用户

  • 企业服务器

  • 物联网设备

  • 软件应用

组件角色关键功能
CA（认证中心）核心权威签发、管理、吊销证书；发布 CRL。
RA（注册中心）辅助/前置受理用户申请、审核身份，转发给 CA。
证书库 / LDAP发布中心存储并分发证书和 CRL，供用户查询下载。
CRL（证书吊销列表）撤销机制（批量）定期列出被吊销的证书序列号。
OCSP（在线证书状态协议）撤销机制（实时）提供实时的证书状态查询服务。
KM / KMS（密钥管理系统）密钥生命周期管理管理密钥的生成、分发、存储、归档、销毁。
密码机 / HSM硬件安全根基物理级保护私钥，执行加密/签名操作，防篡改。

3. 核心概念与原理

3.1 公钥密码学基础

对称加密 vs 非对称加密

| 特性 | 对称加密 | 非对称加密 |

|——|———-|————|

| 密钥数量 | 1个（共享密钥） | 2个（公钥+私钥） |

| 密钥分发 | 困难 | 容易 |

| 计算效率 | 高 | 较低 |

| 典型算法 | AES、DES、3DES | RSA、ECC、ElGamal |

RSA算法

  • 数学基础：大整数分解难题

  • 密钥长度：1024位（已不推荐）、2048位（推荐）、4096位（高安全）

  • 用途：加密、签名、数字信封

ECC（椭圆曲线密码学）

  • 数学基础：椭圆曲线离散对数难题

  • 优势：相同安全级别下密钥更短

  • 常用曲线：P-256、P-384、P-521

3.2 数字签名

签名过程：


原文 → Hash函数 → 摘要 → 私钥加密 → 数字签名

验证过程：


签名 → 公钥解密 → 摘要1

原文 → Hash函数 → 摘要2

       比较摘要1与摘要2

3.3 数字证书

X.509证书结构


Certificate ::= SEQUENCE {

    tbsCertificate       TBSCertificate,

    signatureAlgorithm   AlgorithmIdentifier,

    signatureValue      BIT STRING

}

  

TBSCertificate ::= SEQUENCE {

    version         [0] EXPLICIT Version DEFAULT v1,

    serialNumber         CertificateSerialNumber,

    signature            AlgorithmIdentifier,

    issuer               Name,

    validity             Validity,

    subject              Name,

    subjectPublicKeyInfo SubjectPublicKeyInfo,

    issuerUniqueID  [1] IMPLICIT UniqueIdentifier OPTIONAL,

    subjectUniqueID [2] IMPLICIT UniqueIdentifier OPTIONAL,

    extensions      [3] EXPLICIT Extensions OPTIONAL

}

关键字段说明

| 字段 | 说明 | 示例 |

|——|——|——|

| Serial Number | 证书唯一序列号 | 0x00:8F:E3:D8:2A |

| Subject | 证书持有者 | CN=example.com |

| Issuer | 颁发者 | CN=Root CA, O=Company |

| Valid From | 生效日期 | 2024-01-01 |

| Valid To | 过期日期 | 2025-01-01 |

| Public Key | 公钥信息 | RSA 2048-bit |

| Signature | CA数字签名 | - |

| Extensions | 扩展字段 | SAN、密钥用途等 |

3.4 证书链

定义：从终端实体证书到根CA的证书信任路径。

示例：


Root CA（自签名）

    └── Intermediate CA 1

            └── Intermediate CA 2

                    └── End-Entity Certificate

4.2 证书申请流程

PKCS#10格式：


CertificationRequest ::= SEQUENCE {

    certificationRequestInfo CertificationRequestInfo,

    signatureAlgorithm        AlgorithmIdentifier,

    signature                 BIT STRING

}

  

CertificationRequestInfo ::= SEQUENCE {

    version             INTEGER,

    subject             Name,

    subjectPKInfo       SubjectPublicKeyInfo,

    attributes          [0] Attributes OPTIONAL

}

| 原因码 | 说明 |

|——–|——|

| 1 | 密钥泄露 |

| 2 | CA泄露 |

6.4 证书查询协议

OCSP（Online Certificate Status Protocol）

请求格式：


OCSPRequest ::= SEQUENCE {

    tbsRequest              TBSRequest,

    optionalSignature [0]   EXPLICIT Signature OPTIONAL

}

响应状态：

| 状态 | 含义 |

|——|——|

| Good | 证书有效 |

| Revoked | 证书已吊销 |

| Unknown | CA不识别该证书 |

SCVP（Server-Based Certificate Validation Protocol）

  • 委托证书验证服务

  • 完整验证链校验

8. 安全威胁与防护

8.1 常见威胁

| 威胁类型 | 描述 | 影响 |

|———-|——|——|

| 私钥泄露 | 私钥被窃取或复制 | 身份伪造 |

| CA欺骗 | 攻击者冒充合法CA | 颁发伪造证书 |

| 中间人攻击 | 拦截并篡改通信 | 数据窃取/篡改 |

| 证书欺骗 | 使用伪造证书 | 身份伪造 |

| CRL注入 | 伪造吊销列表 | 拒绝服务 |

| 时间戳篡改 | 修改证书有效期 | 证书滥用 |

8.2 攻击案例

DigiNotar事件（2011）

  • 荷兰CA被入侵

  • 颁发500多张伪造证书

  • 影响Google、Facebook等

StartCom事件（2016）

  • 中国CA被Mozilla吊销

  • 发现违规签发证书

Symantec事件（2018）

  • Google Trust Store移除Symantec证书

  • 理由：信任违规

8.3 防护措施

技术层面：

B. 常用命令

OpenSSL常用命令：


# 生成RSA密钥

openssl genrsa -out private.key 2048

  

# 生成证书请求

openssl req -new -key private.key -out request.csr

  

# 自签名证书

openssl req -x509 -new -nodes -key private.key -sha256 -days 365 -out certificate.crt

  

# 验证证书

openssl x509 -in certificate.crt -text -noout

  

# 验证证书链

openssl verify -CAfile ca.crt certificate.crt

  

# 检查证书过期

openssl x509 -in certificate.crt -noout -dates

  

# 转换证书格式

openssl x509 -in certificate.der -inform DER -out certificate.pem -outform PEM
